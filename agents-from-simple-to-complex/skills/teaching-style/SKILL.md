# Teaching Style

Write code that is:

- simple
- clear
- easy to teach from

Prefer:

- short functions
- direct control flow
- obvious variable names
- minimal abstraction

Avoid unless the user asks for them:

- logging
- special exception handling
- extra indirection
- unnecessary helper layers
